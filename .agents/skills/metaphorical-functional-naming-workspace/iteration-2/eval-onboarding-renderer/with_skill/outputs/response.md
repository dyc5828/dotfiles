# Naming: Cross-Platform Onboarding Flow Renderer

## Core Function

This system takes a single onboarding flow definition and produces platform-specific implementations for web, iOS, Android, and email. The fundamental action is not "onboarding" or "rendering" - it is taking one input and splitting it into multiple distinct, platform-native outputs. It is a one-to-many transformation. A single canonical description fans out into several concrete forms, each shaped to fit its target surface.

There is also a quality of translation happening - the same intent is being faithfully carried across into different material. And a quality of stamping or casting - one master pattern producing many copies in different materials.

## Metaphor Domains

- **Optics** - a single beam of light entering a medium and splitting into a spectrum of distinct wavelengths. Each output is a different expression of the same source.
- **Casting/Metalwork** - a single mold or die used to produce the same shape in different materials. One master pattern, many material outputs.
- **Printing/Reproduction** - a plate or block that transfers an image onto different surfaces. The definition is the plate; the platforms are the surfaces.
- **Weaving/Textiles** - a single pattern or loom setup that produces fabric. The pattern is defined once; the material runs through.
- **Cartography** - a single survey projected onto different map formats. Same territory, different projections.
- **Music** - a single composition arranged for different instruments. Same piece, different voicings.

## Candidate Names

1. **Prism** - a single beam enters, multiple distinct beams exit. The metaphor is precise: one onboarding definition goes in, platform-specific implementations come out, each "refracted" to its target. Immediate, physical, and the one-to-many split is the defining characteristic.

2. **Die** - in manufacturing, a die stamps a single form into whatever material is fed through it. The onboarding definition is the die; each platform is a different material receiving the same shape. Short, punchy, industrial.

3. **Loom** - a loom holds a fixed pattern and weaves thread through it to produce fabric. The definition is the warp; each platform rendering is a pass of the shuttle. Implies structured, repeatable output from a single setup.

4. **Mold** - one master form, poured into with different materials to produce platform-specific results. Very direct metaphor for write-once, render-many.

5. **Plate** - in printing, a plate carries the image and transfers it onto any surface. The onboarding definition is the plate; the platforms are the paper stock. Clean, calm, industrial.

6. **Facet** - each platform output is a different facet of the same gem - a different angle on the same underlying shape. Implies unity with variation.

7. **Shuttle** - the part of a loom that carries thread back and forth across the warp. Implies carrying the same payload across different lanes. Also evokes a vehicle moving between destinations.

## Strongest Candidates

**Prism** is the standout. The metaphor is immediate and requires no explanation - light goes in one side, a spectrum comes out the other. That maps perfectly to "single definition in, multiple platform implementations out." It is short, concrete, visually vivid, and brandable. It does not describe the category, it describes the mechanism. It also carries connotations of clarity and precision, which fits a system that needs to faithfully translate intent across platforms.

**Loom** is strong for a different reason. It emphasizes the structured, repeatable nature of the system - you set up the pattern once and then run material through it. It feels durable and craft-oriented. The "write-once" aspect maps well to setting up the loom; the "render-everywhere" maps to weaving with different threads. It is warm, tactile, and easy to say.

**Die** is the most industrial and precise option. It foregrounds the stamping/manufacturing metaphor - one tool, many outputs. It is extremely short and punchy. The risk is that "die" has an obvious alternate meaning that could create odd sentences in documentation, but as a product name it is distinctive and memorable.
